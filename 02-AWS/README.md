# Phase 1 – AWS Foundations

## Objective

Build a strong foundation in Amazon Web Services (AWS) required for Database Administration, Cloud Infrastructure, High Availability, Security, Performance Tuning, Backup & Recovery, and Disaster Recovery.

This phase establishes the cloud knowledge required before working with Amazon RDS, Amazon Aurora, PostgreSQL, MySQL, SQL Server, and Oracle databases on AWS.

---

# Learning Objectives

By the end of this phase, you will be able to:

- Understand AWS Global Infrastructure
- Explain AWS Regions and Availability Zones
- Configure AWS Identity and Access Management (IAM)
- Launch and manage EC2 instances
- Understand Virtual Private Cloud (VPC)
- Configure Security Groups and Network ACLs
- Manage Elastic Block Store (EBS)
- Monitor AWS resources using CloudWatch
- Use the AWS Command Line Interface (AWS CLI)
- Apply AWS security and cost optimization best practices

---

# Prerequisites

Before starting this phase, ensure:

- Phase 0 (Environment Setup) is completed.
- AWS CLI is configured.
- Git is configured.
- GitHub repository is ready.
- AWS Free Tier account is active.
- IAM User is configured.
- Multi-Factor Authentication (MFA) is enabled.
